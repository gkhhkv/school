# 订单模块需求规格

> 版本: 1.0 | 日期: 2026-05-30 | 状态: 已确认

## 1. 模块概述

订单模块是电商购物平台的核心业务模块，负责管理用户下单、订单查询、状态流转和订单取消的全流程。

## 2. 功能需求

### 2.1 创建订单
- 用户提交订单时，传入商品列表（商品ID + 数量）、收货信息（地址、联系人、电话）
- 系统校验所有商品库存是否充足，不足则拒绝下单
- 生成全局唯一订单编号
- 计算订单总金额（各商品单价×数量之和）
- 扣减对应商品库存
- 订单初始状态为"待付款"

### 2.2 分页查询订单
- 支持按订单状态筛选
- 支持分页参数（页码、每页条数）
- 按创建时间降序排列

### 2.3 查询订单详情
- 根据订单编号查询订单主信息 + 商品明细列表
- 订单不存在时返回明确错误

### 2.4 更新订单状态
- 支持状态流转：待付款→已付款→已发货→已完成
- 已取消和已完成为终态，不可再变更
- 非法状态转换需返回明确错误

### 2.5 取消订单
- 仅"待付款"状态可取消
- 取消后恢复商品库存
- 状态变更为"已取消"

## 3. 订单状态流转

```
待付款 → 已付款 → 已发货 → 已完成
  ↓         ↓
  └──→ 已取消 ←──┘
```

## 4. 非功能需求

- 创建订单和取消订单需保证事务一致性
- 订单编号全局唯一
- 接口遵循 RESTful 规范
- 统一返回格式（code + message + data）
